# Batch Documentation Review Checklist

Use this checklist when reviewing a manifest-driven documentation update.

- Every manifest item points to an existing file.
- Every proposed fix is tied to a reader-visible problem.
- High-severity fixes have a clear validation step.
- Security-sensitive guidance avoids secrets, tokens, and private identifiers.
- The final diff is readable without opening the manifest.
- A human reviewer has accepted or rejected each item.
